# 가상악기 (Virtual Instruments) 요구사항 문서

## 소개

사용자가 PDF 악보를 업로드하여 자동으로 음표를 인식하고, 선택한 가상악기들로 자동 연주할 수 있는 웹 기반 가상악기 시스템입니다. 이 기능은 음악 교육, 연습, 그리고 창작 활동을 지원하는 것을 목표로 합니다.

## 요구사항

### 요구사항 1: PDF 악보 업로드 및 음표 인식

**사용자 스토리:** 음악가로서 PDF 악보를 업로드하여 자동으로 음표 정보를 추출하고 싶습니다.

#### 수락 기준

1. WHEN 사용자가 PDF 파일을 드래그 앤 드롭하거나 파일 선택 버튼을 클릭 THEN 시스템은 PDF 파일을 업로드해야 합니다
2. WHEN PDF 파일이 업로드되면 THEN 시스템은 OCR 처리 진행 상태를 실시간으로 표시해야 합니다
3. WHEN OCR 처리가 완료되면 THEN 시스템은 음표 데이터를 JSON 형식으로 변환해야 합니다
4. IF PDF 파일이 유효하지 않거나 음표 인식에 실패하면 THEN 시스템은 명확한 오류 메시지를 표시해야 합니다
5. WHEN 음표 인식이 완료되면 THEN 시스템은 인식된 음표 정보(음 높이, 박자, 길이, 템포, 마디)를 사용자에게 미리보기로 제공해야 합니다

### 요구사항 2: 가상악기 선택 및 관리

**사용자 스토리:** 연주자로서 피아노, 기타, 드럼 중에서 원하는 악기를 선택하여 합주 형태로 연주하고 싶습니다.

#### 수락 기준

1. WHEN 사용자가 악기 선택 화면에 접근 THEN 시스템은 피아노, 기타, 드럼 선택 옵션을 제공해야 합니다
2. WHEN 사용자가 악기를 선택 THEN 시스템은 다중 선택을 허용해야 합니다
3. WHEN 악기가 선택되면 THEN 시스템은 해당 악기의 사운드 라이브러리를 로드해야 합니다
4. WHEN 사용자가 악기 선택을 변경 THEN 시스템은 실시간으로 선택 상태를 업데이트해야 합니다
5. WHEN 복수의 악기가 선택되면 THEN 시스템은 합주 형태의 동시 재생을 지원해야 합니다

### 요구사항 2.1: 드럼 시스템 구현

**사용자 스토리:** 드러머로서 MusicXML이나 피아노롤 데이터에서 드럼 파트를 인식하고 실제 드럼 사운드로 재생하고 싶습니다.

#### 수락 기준

1. WHEN 드럼이 선택되면 THEN 시스템은 Tone.js 기반으로 드럼 샘플을 로드해야 합니다
2. WHEN 드럼 샘플을 로드할 때 THEN 시스템은 다음 URL에서 실제 존재하는 샘플을 사용해야 합니다:
   - kick: https://upload.wikimedia.org/wikipedia/commons/4/45/Kick_1.wav
   - snare: https://upload.wikimedia.org/wikipedia/commons/7/70/Snare_1.wav
   - hihat_closed: https://upload.wikimedia.org/wikipedia/commons/8/89/Closed-Hi-Hat.wav
   - hihat_open: https://upload.wikimedia.org/wikipedia/commons/3/3b/Open-Hi-Hat.wav
   - tom: https://upload.wikimedia.org/wikipedia/commons/6/6f/Tom-Drum.wav
   - crash: https://upload.wikimedia.org/wikipedia/commons/5/5c/Cymbal_Crash.wav
3. WHEN 피아노롤 데이터에서 드럼 노트를 처리할 때 THEN 시스템은 다음 MIDI 매핑을 사용해야 합니다:
   - MIDI 36 → kick
   - MIDI 38 → snare
   - MIDI 42 → hihat_closed
   - MIDI 46 → hihat_open
   - MIDI 45 → tom
   - MIDI 49 → crash
4. WHEN 공통 악보 파트(P1)에 드럼 노트가 포함되어 있으면 THEN 시스템은 멜로디 악기 대신 드럼 샘플러로 재생해야 합니다
5. WHEN 드럼 샘플 재생에 실패하면 THEN 시스템은 콘솔에 오류를 출력하고 계속 진행해야 합니다
6. WHEN 드럼 시스템이 동작할 때 THEN 기존 기타/피아노 시스템과 충돌 없이 분리되어 동작해야 합니다
7. WHEN 향후 샘플을 로컬 파일로 교체할 때 THEN 시스템은 쉽게 URL을 변경할 수 있도록 구조화되어야 합니다

### 요구사항 3: 자동 연주 엔진

**사용자 스토리:** 음악 학습자로서 업로드한 악보를 자동으로 연주하여 듣고 학습하고 싶습니다.

#### 수락 기준

1. WHEN 사용자가 재생 버튼을 클릭 THEN 시스템은 선택된 악기들로 자동 연주를 시작해야 합니다
2. WHEN 자동 연주가 진행 중 THEN 시스템은 현재 재생 위치와 마디를 실시간으로 표시해야 합니다
3. WHEN 사용자가 일시정지 버튼을 클릭 THEN 시스템은 현재 위치에서 재생을 일시정지해야 합니다
4. WHEN 사용자가 정지 버튼을 클릭 THEN 시스템은 재생을 중단하고 처음 위치로 돌아가야 합니다
5. WHEN 사용자가 템포 슬라이더를 조절 THEN 시스템은 40-200 BPM 범위에서 재생 속도를 실시간으로 조절해야 합니다
6. WHEN 재생이 완료되면 THEN 시스템은 자동으로 처음 위치로 돌아가야 합니다

### 요구사항 4: 사운드 품질 및 성능

**사용자 스토리:** 음악가로서 고품질의 사운드로 자연스러운 연주를 듣고 싶습니다.

#### 수락 기준

1. WHEN 피아노가 선택되면 THEN 시스템은 샘플 기반 또는 SoundFont를 사용하여 리얼한 피아노 사운드를 제공해야 합니다
2. WHEN 기타가 선택되면 THEN 시스템은 어쿠스틱 또는 일렉 기타 SoundFont를 사용해야 합니다
3. WHEN 드럼이 선택되면 THEN 시스템은 URL 기반 실제 드럼 샘플을 사용해야 합니다
4. WHEN 여러 악기가 동시에 재생 THEN 시스템은 지연 없이 동기화된 재생을 제공해야 합니다
5. WHEN 사용자가 웹 브라우저에서 접근 THEN 시스템은 WebAudio API 또는 Tone.js를 사용하여 안정적인 오디오 재생을 제공해야 합니다

### 요구사항 5: 사용자 인터페이스

**사용자 스토리:** 사용자로서 직관적이고 사용하기 쉬운 인터페이스를 통해 가상악기 기능을 이용하고 싶습니다.

#### 수락 기준

1. WHEN 사용자가 가상악기 탭에 접근 THEN 시스템은 명확하게 구분된 섹션들을 표시해야 합니다
2. WHEN 파일 업로드 영역이 표시 THEN 시스템은 드래그 앤 드롭과 클릭 업로드를 모두 지원해야 합니다
3. WHEN OCR 처리 중 THEN 시스템은 진행률과 현재 상태를 시각적으로 표시해야 합니다
4. WHEN 재생 컨트롤이 표시 THEN 시스템은 재생/일시정지/정지 버튼과 진행 바를 제공해야 합니다
5. WHEN 템포 조절 슬라이더가 표시 THEN 시스템은 현재 BPM 값을 실시간으로 표시해야 합니다
6. WHEN 모바일 기기에서 접근 THEN 시스템은 반응형 디자인으로 최적화된 인터페이스를 제공해야 합니다

### 요구사항 6: 오류 처리 및 사용자 피드백

**사용자 스토리:** 사용자로서 시스템 오류나 문제 상황에 대해 명확한 피드백을 받고 싶습니다.

#### 수락 기준

1. WHEN PDF 파일 형식이 지원되지 않으면 THEN 시스템은 지원되는 형식을 안내해야 합니다
2. WHEN 음표 인식에 실패하면 THEN 시스템은 실패 원인과 해결 방법을 제시해야 합니다
3. WHEN 오디오 재생에 문제가 발생하면 THEN 시스템은 브라우저 호환성 정보를 제공해야 합니다
4. WHEN 파일 크기가 제한을 초과하면 THEN 시스템은 최대 허용 크기를 안내해야 합니다
5. WHEN 네트워크 오류가 발생하면 THEN 시스템은 재시도 옵션을 제공해야 합니다

### 요구사항 7: 성능 및 최적화

**사용자 스토리:** 사용자로서 빠르고 안정적인 성능으로 가상악기 기능을 이용하고 싶습니다.

#### 수락 기준

1. WHEN 사운드 라이브러리를 로드 THEN 시스템은 5초 이내에 로딩을 완료해야 합니다
2. WHEN OCR 처리를 수행 THEN 시스템은 일반적인 악보(2-4페이지)를 30초 이내에 처리해야 합니다
3. WHEN 오디오 재생을 시작 THEN 시스템은 100ms 이내의 지연시간을 유지해야 합니다
4. WHEN 여러 악기가 동시 재생 THEN 시스템은 CPU 사용률을 70% 이하로 유지해야 합니다
5. WHEN 메모리 사용량이 증가 THEN 시스템은 적절한 가비지 컬렉션으로 메모리 누수를 방지해야 합니다
